Ensemble Mimarisi Detayı (Stage 1)

Katman A — Kural Motoru: 58 re.compile regex, First-Match-Wins stratejisi, Specific→General sıralama, Precision = %100

Katman B — Turkish ELECTRA: dbmdz/electra-small-turkish-cased-discriminator, Sub-word WordPiece tokenizasyon, Solo F1 = 0.875

Katman C — TF-IDF + Kalibre SVC: 30K kelime (ngram 1-3) + 20K karakter (ngram 2-4/5) özelliği, C=0.5, CalibratedClassifierCV (Platt scaling)

Ensemble Formülü: B×(1−α) + C×α, α=0.6 → Makro F1 = 0.9282 | Conf. Threshold = 0.5 (üretimde 0.65)

Tersine Mühendislik Test Stratejisi

Literatürde eşine az rastlanan kör test (blind test) stratejisi:

• bankaverison.xlsx "Diğer" kategorilerinin %80'i test setine saklandı
• bankaverison.xlsx "Sağlık" kategorisinin %100'ü test setine saklandı
• YENİVERİTEST.xlsx'in %100'ü doğrudan test setine aktarıldı

Sonuç: Model %95 sentetik veriyle eğitilip neredeyse tamamen GERÇEK veriyle test edildi. Bu, modelin sentetik veriyi ezberlemediğinin (overfitting yapmadığının) en güçlü matematiksel kanıtıdır.

Veri Temizleme Pipeline'ı

  • 271.352 satır ham sentetik veriden drop_duplicates() ile tekrarlar silindi
  • Gerçek test metinleriyle örtüşen sentetik satırlar ~isin(real_lower) ile kaldırıldı (Data Leakage önlemi)
  • 3 karakterden kısa anlamsız metinler (Örn: "su", "al") algoritma ile dışlandı
  • Türkçe karakter uyumlu lowercase: I, İ, Ü, Ş, Ö, Ç tam destekli
  • Regex [^\w\s] ile emoji, özel karakter, fazladan boşluk temizlendi
  • StandardScaler / Min-Max Scaler sadece Train setine fit edildi (test sadece transform)
  • IsolationForest(n_estimators=200, contamination=0.05) ile 158 anomali (%5.02) otomatik tespit edildi
  • Eğitim/Test ayrımından önce veri klonlama yapıldı (leakage bloğu)

Sentetik Veri Üretim Yöntemleri

NLP Verisi (Stage 1)

GPT-4 Prompt Engineering + Python Faker kullanıldı. Kısaltmalar, POS cihazı varyasyonları ve kasıtlı yazım hataları prompt'a dahil edilerek gerçek banka diline yaklaştırıldı. Ham 271K satırdan deduplication sonrası 95K satır elde edildi. Her sınıfın eğitim setinde minimum 4.000+ örneği bulunuyor.

Koçluk Verisi (Stage 4 — SMOTE)

NEEDS_BUDGETING azınlık profillerinin finansal metriklerine ±%5 Gaussian Noise eklenerek 1.965 adet sentetik klon üretildi. Bu hamle tek başına XGBoost F1 skorunu %67 → %93.6'ya çıkardı. Gaussian Noise sayesinde overfitting oluşmadı.

Model Parametreleri — Tam Döküm

TF-IDF Vektörizasyon

  • Kelime: max_features=30.000, ngram_range=(1,3)
  • Karakter: max_features=20.000, ngram_range=(2,4 veya 2,5)
  • min_df=5 (nadir kelimeleri eliyor)
  • Stopword kullanılmadı (finansal bankacılık terminolojisi bozulmasın)

LinearSVC + Kalibrasyon

  • C=0.5 (Grid Search ile belirlendi)
  • max_iter=4000, class_weight='balanced'
  • CalibratedClassifierCV (Platt scaling / sigmoid)
  • Loss: Hinge Loss, Multi-class: One-vs-Rest
  • Inference: 2-4ms/işlem

Turkish ELECTRA

  • dbmdz/electra-small-turkish-cased-discriminator
  • Türkçe wikipedia üzerinde önceden eğitilmiş (pre-trained)
  • Sub-word WordPiece tokenizasyon
  • Epoch=3, Batch Size=32, Optimizer=AdamW
  • Learning Rate=1e-5, weight_decay=0.02
  • Dropout=0.3 (overfitting önlemi)
  • Loss: Cross Entropy | Donanım: Colab T4 GPU ~40dk
  • Model boyutu: 53.24 MB

OOD Güven Mekanizması

  • Threshold = 0.65 (P-R eğrisinden F1 maksimum noktası)
  • Altında → DİĞER sınıfına (Güvenli Bekletme)
  • 0.50: yüksek False Positive riski
  • 0.80: yüksek False Negative riski
  • 0.65: F1 tatlı noktası (sweet spot)

Projenin Özgün Katkıları (Literatüre Katkı)

  • Hibrit Mimari (Türkçe Finans NLP'de İlk): Regex + LinearSVC + ELECTRA hiyerarşik pipeline. Literatürdeki saf DL sistemlerine kıyasla %40+ hızlı, %30 daha ucuz sunucu maliyeti
  • Türkçe Finansal NLP Boşluğu Dolduruldu: Türkçe eklemeli dil yapısına özel ELECTRA ile TÜBİTAK/ulusal alanda kritik boşluk kapatıldı
  • Tersine Mühendislik Test Stratejisi: Gerçek verinin %80-100'ü teste ayrıldı — overfitting olmadığının matematiksel kanıtı
  • SHAP Açıklanabilirlik (XAI): "Kara Kutu" problemi çözüldü — koçluk kararları net SHAP değerleriyle şeffaflaştırıldı
  • OOD Güvenlik Duvarı: %65 threshold altındaki işlemleri reddederek yanlış bilgi vermek yerine dürüstçe "DİĞER" kategorisine alır
  • Döngüsel Zaman Kodlaması: Ay sayısı yerine month_sin/month_cos — takvim sürekliğini modele doğru öğretiyor. SHAP'ta 2. sıraya oturdu
  • HIGH_DEBT_RISK Recall = %95.5: Finansal krizi olan kullanıcıların %95.5'i yakalanıyor — bankacılık için kritik güvenlik ağı
  • Sıfır Maliyetli Ön Filtre: %31.2'lik yükü Regex ile bedavaya çözüyor — AWS/Azure sunucu maliyetlerinde %30 tasarruf

Kapsamlı Jüri Soru-Cevap Kataloğu

S1: Gerçek veri %5.3, sentetik %94.7 — model güvenilir mi?

Veri anonimleştirme ve KVKK kısıtları nedeniyle sentetik yaklaşım seçildi. En kritik nokta: Test seti büyük çoğunlukla gerçek veriden oluştu (Tersine Mühendislik). Sentetik verinin istatistiksel dağılımı, metin uzunlukları, banka kısaltmaları ve marka varyasyonları gerçeğe birebir benzetildi. 5-Fold CV ile overfitting matematiksel olarak dışlandı.

S2: Sentetik veri olmadan model F1 kaç? (Ablation)

Stage 4 XGBoost koçluk F1'i %67'lerde kalıyor. NEEDS_BUDGETING azınlık sınıfı yeterince temsil edilemiyor. Gaussian Noise ile 1.965 klon eklenmesi sonrasında %93.6'ya çıktı (+%26.6 artış). Bu projenin en büyük bilimsel katkısının "Veri Dengesizliği Çözümü" olduğunu kanıtlıyor.

S3: MAPE %75.12 çok yüksek değil mi?

Stage 3 amacı TL bazında kesin tahmin değil, harcama eğilimi/risk sinyali tespiti (Trend Analysis). Sıfıra yakın harcama aylarında MAPE matematiksel olarak şişer (bölende küçük değer). WAPE = %29.16 ile sistem ağırlıklı bazda güvenilirdir. Moving Average baseline karşılaştırması rapora eklenecek.

S4: LinearSVC normalde predict_proba vermez — güven skoru nasıl üretildi?

CalibratedClassifierCV kullanılarak Platt scaling (sigmoid calibration) ile SVC marjin (hyperplane mesafe) değerleri güvenilir olasılığa dönüştürüldü. Threshold 0.65, Precision-Recall eğrisinin F1 maksimum noktasından seçildi.

S5: ELECTRA üretimde aktif mi?

Üretim öncelikli hat: Regex + Kalibre SVC (2-4ms). ELECTRA, karmaşık bağlamlarda ikincil (fallback) model. Maliyet ve latency nedeniyle her işlemde çalıştırılmıyor. Bu bilinçli bir mühendislik kararı — literatürdeki farkımız bu noktada başlıyor.

S6: K-Means Silhouette 0.284 düşük değil mi?

Finansal müşteri profilleri doğaları gereği net sınırlarla ayrılmaz (overlapping). Calinski-Harabasz = 2943.11 (çok yüksek — kümelerin birbirinden mükemmel ayrıştığını kanıtlar) ve Davies-Bouldin = 1.5588 ile birlikte değerlendirildiğinde kümelerin finansal iş anlamı oluşturduğu kanıtlandı.

S7: XGBoost basit If-Else kuralı mı öğreniyor?

Basit kural: gelir < gider → uyarı. XGBoost ise "geliri yüksek olsa bile son 3 aylık harcama eğilimi kontrolsüz artıyor" gibi doğrusal olmayan (non-linear) etkileşimleri öğrendi. SHAP analizi kararların net_cashflow ve savings_rate gibi finansal iş mantığıyla örtüştüğünü kanıtladı.

S8: Sistem üretime hazır mı?

Evet. FastAPI + React olarak deploy edildi. 2-4ms inference. Demo modu (ML bağımlılığı olmadan), gerçek CSV/Excel yükleme, senaryo simülasyonu (5 kaydırıcı), PDF/Excel rapor dışa aktarım tamamlandı. GPU zorunluluğu yok. Docker + Kafka bir sonraki aşama.

S9: KVKK uyumu nasıl sağlandı?

kvkk_masker.py tamamen pattern-based çalışır. Hardcoded isim listesi yoktur. Banka ekstrelerinde kişi adları BÜYÜK HARF yazılır — bu prensipten yararlanılarak _NAME_BEFORE_BANK regex'i büyük/küçük harfe duyarlı derlendi. "Ziraat Mobil" gibi banka isimleri yanlışlıkla maskelenmez. 7 KVKK testi geçildi.

S10: Veri sızıntısını nasıl engellediniz?

Veri klonlama/çoğaltma işleminden ÖNCE train/test ayrımı yapıldı. Gerçek test metinleri ve n-gram benzerlikleri sentetik eğitim verisinden ~isin(real_lower) ile izole edildi. StandardScaler yalnızca train setine fit edildi. YENİVERİTEST.xlsx %100 test setine ayrıldı.

En Güçlü 5 Savunma Noktası

1. HIGH_DEBT_RISK Recall = %95.52 → Krizdeki kullanıcıların %95.52'si yakalanıyor — bankacılık için kritik güvenlik ağı
2. SAGLIK F1 = 1.000 (Zero-Defect) → Sentetik klonlama + Stratified Split'in azınlık sınıfa katkısı kanıtlandı
3. Ablation: Sentetik veri olmadan F1 %67 → Sentetikle %93.6 → +%26.6, en büyük bilimsel katkı
4. 5-Fold CV std = ±%0.77 → Model kararlı ve üretim ortamına alınabilir
5. Regex %31.2 iş yükü sıfır maliyetle çözüldü → Operasyonel üstünlük ve bulut tasarrufu
